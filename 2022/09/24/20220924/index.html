  • 再不写东西，我博客就要长草了T_T。

ICM指的是Image Compression For Machine，VCM指的是Video Compression For Machine。“VCM”这个英文缩写是MPEG近期成立的一个新的标准小组，探索I/V compression for Machine。
关于VCM相关工作的介绍->传送门。VCM Standord官网

VCM标准与综述

在"Video Coding for Machines: A Paradigm of Collaborative Compression and Intelligent Analytics"这篇论文中，VCM的组织成立者——北大某chen姓大牛组概括了VCM目前的主要几个任务：

  1. Machine vision targeted codec
  2. Intermediate feature compression
  3. Intermediate feature optimization
  4. Features assisted coding
  5. Scalable coding

Machine vision targeted codec

这个任务是我目前想做的方向，它其实是图片/视频压缩发展到一定阶段必然会有的一种想法。拿目前端到端的图片编码为例，当前的图片编码以PSNR或者MS-SSIM作为Distortion得到衡量指标，并作为loss项参与到端到端的训练中去，使得网络能够在训练过程中减少图像压缩的像素级和分布级的失真，从而满足人类视觉的需求。但其实在当前的网络传输中，很多数据其实是从Machine To Machine的，所以能不能用Machine所需要的失真度量来优化网络呢？这也就是改任务的想法。

MVTC任务结构
需要注意的是Machine vision targeted codec不一定要求的是端到端的编码结构，只要能够达到Task上的提升都算做一种方法。MVTC具体来说又可以分为三类：
  1. Side Information
  2. Machine vision constraint training(End To End)
  3. Semantic information preservation(没有做太多调研)

Side Information我认为大多数是从视觉任务的网络上获得某些对视觉任务很重要的信息，作为边信息指导一个传统编码方法如Hevc，VVC等对图片/视频进行编码，与其相关的两篇论文为：

  • A Novel Video Coding Strategy in HEVC for Object Detection(Video)
  • Visual analysis motivated rate-distortion model for image coding(Image)
